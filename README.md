# Следящая рамка

Веб-приложение: MediaPipe Pose находит крайние точки тела — ладони, ступни, верх головы — и по ним строит рамку кадра. Линии рамки тянутся до краёв холста, снаружи остаётся чистый фон, по периметру идёт подпись. Рамка плавно следует за движением.

## Файлы

- `index.html` — разметка и панель управления
- `style.css` — стили (IBM Plex Mono, бумажный фон)
- `app.js` — детекция позы, расчёт рамки, отрисовка, запись

Сборка не нужна: MediaPipe Tasks Vision подключается с CDN (jsdelivr), модель `pose_landmarker_lite` грузится с Google при первом запуске.

## Запуск локально

ES-модули не работают через `file://`, нужен любой локальный сервер:

```
npx serve .
# или
python3 -m http.server 8000
```

Камера требует HTTPS или `localhost` — на локальном сервере и на GitHub Pages всё работает.

## Деплой на GitHub Pages

1. Создать репозиторий, залить три файла в корень.
2. Settings → Pages → Source: Deploy from a branch → `main` / `(root)`.
3. Через минуту приложение доступно по адресу `https://<username>.github.io/<repo>/`.

## Управление

- **Камера / Видеофайл** — источник. Файл зациклен, удобно для архивных фрагментов.
- **Отступ** — поля рамки вокруг крайних точек (в % от кадра).
- **Плавность** — инерция рамки: слева — резко следует, справа — вязко плывёт.
- **Подпись** — текст по периметру рамки (по умолчанию как на референсе).
- **Фон** — белый «бумажный» или чёрный.
- **Запись** — пишет холст в `tracking-frame.webm` (VP9, 30 fps, ~12 Мбит/с).

## Конвертация записи в ProRes

Для медиасервера или монтажа webm удобно перегнать в ProRes 422 LT:

```
ffmpeg -i tracking-frame.webm -c:v prores_ks -profile:v 1 -pix_fmt yuv422p10le out.mov
```

## Как считается рамка

Берутся точки MediaPipe Pose: запястья и пальцы (15–22), лодыжки, пятки, носки (27–32). Верх головы модель не даёт, поэтому он оценивается: от линии глаз вверх на ~0.9 расстояния между ушами. Точки с visibility < 0.4 отбрасываются. По оставшимся берётся min/max по X и Y, добавляется отступ, результат сглаживается экспоненциальной интерполяцией.

## Идеи для развития

- Зеркалить веб-камеру (отразить и видео, и координаты рамки).
- Несколько людей — `numPoses > 1`, отдельная рамка на каждого.
- Экспорт в фиксированном разрешении (offscreen-canvas 1080×1350 и т.п.).
- Режим «только линии» без видео внутри — силуэтная версия для проекции.
